## Stage 1 — Programming Fundamentals (Python, Git, Jupyter)

**Why it matters:** Python and Git are used daily to manipulate data, train models, and collaborate. Strong fundamentals prevent you from getting stuck later on core language or version-control issues instead of the actual data science.

**Core topics:** Python syntax & control flow · Data structures (lists, dicts, sets, tuples) · Functions & modules · Jupyter Notebook / VS Code · Git & GitHub (commits, branches, PRs)

| Resource | Platform | Duration | Level |
|---|---|---|---|
| [Data Analysis with Python](https://www.freecodecamp.org/learn/data-analysis-with-python/) 🥇 | freeCodeCamp | ~120–150 hrs | Beginner→Intermediate |
| [CS50's Intro to Programming with Python](https://cs50.harvard.edu/python) 🥈 | Harvard (CS50) | ~10 weeks | Beginner |
| [Python for AI and Data Science](https://freeacademy.ai/blog/best-free-python-for-ai-courses-2026) 🏆 | FreeAcademy.ai | ~30–40 hrs | Beginner→Intermediate |
| [Kaggle Learn — Python](https://www.kaggle.com/learn/python) | Kaggle | 3–5 hrs | Beginner |
| [The Python Tutorial](https://docs.python.org/3/tutorial/) 📚 | python.org (docs) | Reference | Intermediate |

**Learning order:** CS50P (or Python for Everybody) → freeCodeCamp Python/Data Analysis sections → Kaggle Python micro-course → Git & GitHub basics.

**⚠️ Common mistakes:** Lingering on syntax without touching data libraries · skipping Git early · juggling too many courses at once.

**Project ideas:** CLI expense tracker (beginner) · CSV summary-stats notebook (intermediate) · Python package wrapping a public API (advanced).

**⏱ Estimated time:** 60–100 hrs · 3–5 weeks

## Stage 2 — Mathematics for Data Science

**Why it matters:** Math explains *why* models behave the way they do — gradients, regularization, overfitting, and uncertainty.

**Core topics:** Linear algebra (vectors, matrices, eigenvalues) · Probability & distributions · Statistics (hypothesis testing, confidence intervals) · Calculus basics (derivatives, gradients)

| Resource | Platform | Duration | Level |
|---|---|---|---|
| [Mathematics for Machine Learning Specialization](https://www.coursera.org/specializations/mathematics-machine-learning) 🥇 *(audit free)* | Coursera / Imperial College London | ~12–18 weeks | Foundational–Intermediate |
| [Khan Academy — Statistics & Probability](https://www.khanacademy.org/math/statistics-probability) / [Linear Algebra](https://www.khanacademy.org/math/linear-algebra) 🥈 | Khan Academy | 40–60 hrs | Beginner |
| [StatQuest with Josh Starmer](https://www.youtube.com/@statquest) 🥈 | YouTube | Flexible | Beginner–Intermediate |
| [Machine Learning Crash Course](https://developers.google.com/machine-learning/crash-course) 🥉 | Google Developers | Few hours | Beginner–Intermediate |

**Learning order:** Khan Academy stats/probability (parallel to Stage 1) → Imperial Math for ML (audit), Linear Algebra → Calculus → PCA → StatQuest whenever a concept is unclear.

**⚠️ Common mistakes:** Trying to master all pure math before touching ML · ignoring probability · not coding what you learn.

**Project ideas:** Simulate dice rolls vs. theoretical probability (beginner) · Linear regression from scratch with gradient descent (intermediate) · PCA from scratch vs. scikit-learn (advanced).

**⏱ Estimated time:** 80–120 hrs · 4–8 weeks

## Stage 3 — Data Analysis (NumPy, Pandas, Cleaning, EDA)

**Why it matters:** Most real data science work is cleaning, joining, and exploring data — not fancy modeling.

**Core topics:** NumPy arrays & operations · Pandas (Series, DataFrames, groupby, joins) · Data cleaning & missing data · Exploratory Data Analysis (EDA)

| Resource | Platform | Duration | Level |
|---|---|---|---|
| [Data Analysis with Python](https://www.freecodecamp.org/learn/data-analysis-with-python/) 🥇 | freeCodeCamp | ~120–150 hrs | Beginner→Intermediate |
| [Kaggle Learn — Pandas](https://www.kaggle.com/learn/pandas) 🥈 | Kaggle | 3–5 hrs | Beginner |
| [Kaggle Learn — Data Cleaning](https://www.kaggle.com/learn/data-cleaning) 🥉 | Kaggle | Few hours | Beginner |
| *Python for Data Analysis* by Wes McKinney 🏆 | Book + notebooks | ~20–30 hrs | Intermediate |
| [pandas docs](https://pandas.pydata.org/docs) / [NumPy docs](https://numpy.org/doc) 📚 | Official documentation | Reference | Intermediate |

**⚠️ Common mistakes:** Over-relying on Excel instead of pandas · treating EDA as "just plotting" · ignoring data types (object vs numeric vs datetime).

**Project ideas:** EDA notebook on a public dataset (beginner) · Data-cleaning pipeline for raw CSVs (intermediate) · Automated data-quality report generator (advanced).

**⏱ Estimated time:** 40–80 hrs · 2–5 weeks

## Stage 4 — Data Visualization

**Why it matters:** Visualization is how you communicate findings to stakeholders — and how you debug data intuitively.

**Core topics:** Matplotlib basics · Seaborn (statistical plots) · Plotly (interactive charts) · Storytelling & dashboards

| Resource | Platform | Duration | Level |
|---|---|---|---|
| [Kaggle Learn — Data Visualization](https://www.kaggle.com/learn/data-visualization) 🥈 | Kaggle | 3–5 hrs | Beginner |
| Data Analysis with Python (Matplotlib/Seaborn sections) 🥇 | freeCodeCamp | Part of Stage 1 | Beginner |
| [Storytelling with Data](https://www.storytellingwithdata.com/) 🏆 | Blog | Ongoing | All levels |
| [Matplotlib](https://matplotlib.org/stable/contents.html) / [Seaborn](https://seaborn.pydata.org) / [Plotly](https://plotly.com/python) docs 📚 | Official documentation | Reference | Intermediate |

**⚠️ Common mistakes:** Using default plots with no labels/titles · overloaded dashboards · charts not tied to real business questions.

**Project ideas:** Sales/weather histograms with Seaborn (beginner) · Interactive Plotly dashboard (intermediate) · Data-story blog post (advanced).

**⏱ Estimated time:** 20–40 hrs · 2–4 weeks

## Stage 5 — SQL & Databases

**Why it matters:** Data lives in relational databases at most companies — SQL is the universal query language.

**Core topics:** SELECT/WHERE/GROUP BY/HAVING · JOINs · Subqueries & window functions · PostgreSQL/MySQL basics · Database design & warehousing

| Resource | Platform | Duration | Level |
|---|---|---|---|
| freeCodeCamp — Relational Database & SQL 🥇 | freeCodeCamp | ~60–80 hrs | Beginner |
| [SQLBolt](https://sqlbolt.com/) 🥈 | Interactive lessons | Few hours | Beginner |
| [Kaggle Learn — Intro to SQL](https://www.kaggle.com/learn/intro-to-sql) / [Advanced SQL](https://www.kaggle.com/learn/advanced-sql) 🥉 | Kaggle | Few hours | Beginner–Intermediate |
| [Mode SQL Tutorial](https://mode.com/sql-tutorial/) 🏆 | Mode Analytics | Self-paced | Intermediate |
| [PostgreSQL docs](https://www.postgresql.org/docs) / [MySQL docs](https://dev.mysql.com/doc) 📚 | Official documentation | Reference | Intermediate |

**⚠️ Common mistakes:** Not practicing joins/aggregations enough · learning only one SQL dialect · ignoring query optimization (indexes, subqueries).

**Project ideas:** Small inventory database (beginner) · ETL script loading CSVs into SQL with analytics queries (intermediate) · E-commerce data-warehouse schema (advanced).

**⏱ Estimated time:** 40–60 hrs · 3–5 weeks

## Stage 6 — Machine Learning (Core Algorithms)

**Why it matters:** This is the heart of data science — building, evaluating, and understanding predictive models.

**Core topics:** Supervised learning (regression & classification) · Decision Trees, Random Forests, SVM · Unsupervised learning (k-Means, hierarchical clustering) · Train/test split, cross-validation, metrics, overfitting

| Resource | Platform | Duration | Level |
|---|---|---|---|
| [Machine Learning Specialization](https://www.coursera.org/specializations/machine-learning-introduction) (Andrew Ng) 🥇 *(audit free)* | Coursera / DeepLearning.AI | ~94 hrs (2 months) | Beginner |
| [Machine Learning Crash Course](https://developers.google.com/machine-learning/crash-course) / [ML for Beginners](https://github.com/microsoft/ML-For-Beginners) 🥈 | Google / Microsoft | ~12 weeks | Beginner |
| [Kaggle — Intro to ML](https://www.kaggle.com/learn/intro-to-machine-learning) & [Intermediate ML](https://www.kaggle.com/learn/intermediate-machine-learning) 🥉 | Kaggle | Few hours each | Beginner–Intermediate |
| [Machine Learning Zoomcamp](https://github.com/DataTalksClub/machine-learning-zoomcamp) 🏆 | DataTalks.Club | ~4 months | Beginner–Intermediate |
| [scikit-learn docs](https://scikit-learn.org/stable) 📚 | Official documentation | Reference | Intermediate |

**⚠️ Common mistakes:** Jumping to deep learning before mastering core ML · tracking only accuracy (ignoring precision/recall/F1/ROC-AUC) · skipping cross-validation.

**Project ideas:** House-price prediction (beginner) · Telecom churn model with feature engineering (intermediate) · End-to-end ML API with FastAPI + Docker (advanced).

**⏱ Estimated time:** 100–160 hrs · 6–10 weeks

## Stage 7 — Feature Engineering

**Why it matters:** Good features often matter more than a fancier algorithm.

**Core topics:** Missing-value handling · Categorical encoding (one-hot, target, mean) · Scaling · Feature selection & PCA · Preventing data leakage

| Resource | Platform | Duration | Level |
|---|---|---|---|
| [Kaggle Learn — Feature Engineering](https://www.kaggle.com/learn/feature-engineering) 🥇 | Kaggle | Few hours | Intermediate |
| Microsoft ML for Beginners (feature-engineering lessons) 🥈 | GitHub curriculum | Self-paced | Beginner |
| Machine Learning in Python with scikit-learn (FUN MOOC) 🏆 | Inria / FUN-MOOC | ~36 hrs | Intermediate |
| scikit-learn docs — preprocessing & pipelines 📚 | Official documentation | Reference | Intermediate |

**⚠️ Common mistakes:** Preprocessing train/test separately (data leakage) · label-encoding unordered categories · forgetting to scale for SVM/k-NN.

**Project ideas:** Add engineered ratio/log features to a housing model (beginner) · Feature pipeline with `ColumnTransformer` for churn (intermediate) · PCA + clustering for anomaly detection (advanced).

**⏱ Estimated time:** 30–60 hrs · 3–5 weeks

## Stage 8 — Advanced Machine Learning (XGBoost, LightGBM, Tuning)

**Why it matters:** Gradient-boosted trees dominate tabular-data competitions and real-world production models.

**Core topics:** Gradient boosting (XGBoost, LightGBM, CatBoost) · Hyperparameter tuning (grid/random/Bayesian) · Cross-validation strategies · Ensembles (bagging, stacking, blending)

| Resource | Platform | Duration | Level |
|---|---|---|---|
| Kaggle Intermediate ML + [XGBoost docs](https://xgboost.readthedocs.io) 🥇 | Kaggle / Official docs | Self-paced | Intermediate |
| Microsoft ML for Beginners — advanced lessons 🥈 | GitHub curriculum | Self-paced | Beginner–Intermediate |
| ML Zoomcamp advanced modules & Inria scikit-learn MOOC 🏆 | DataTalks.Club / FUN-MOOC | Self-paced | Intermediate |
| [LightGBM docs](https://lightgbm.readthedocs.io) 📚 | Official documentation | Reference | Intermediate |

**⚠️ Common mistakes:** Overfitting with deep trees / too many boosting rounds · skipping cross-validation · ignoring class imbalance.

**Project ideas:** Swap Random Forest for XGBoost in the house-price model (beginner) · Tuned XGBoost credit-risk model (intermediate) · Ensemble of tree, boosting, and neural models for a Kaggle competition (advanced).

**⏱ Estimated time:** 40–70 hrs · 3–6 weeks

## Stage 9 — Deep Learning

**Why it matters:** Deep learning powers NLP, vision, speech, and generative AI — and is increasingly relevant even for tabular work.

**Core topics:** Neural network basics (layers, activations, backprop) · CNNs · RNN/LSTM/GRU · PyTorch & TensorFlow/Keras basics · Training loops, regularization, optimizers

| Resource | Platform | Duration | Level |
|---|---|---|---|
| [Practical Deep Learning for Coders](https://course.fast.ai/) 🥇 | Fast.ai | ~13.5 hrs + coding | Beginner-friendly |
| Machine Learning with Python (PyTorch) 🥈 | freeCodeCamp | Tens of hours | Intermediate |
| Kaggle CV / NLP competition notebooks 🏆 | Kaggle | Self-paced | Intermediate |
| [PyTorch docs](https://pytorch.org/docs) / [Keras docs](https://keras.io) 📚 | Official documentation | Reference | Intermediate |

**⚠️ Common mistakes:** Training deep models on tiny datasets without regularization · ignoring proper train/val/test splits · not monitoring loss/metrics or using early stopping.

**Project ideas:** Cats-vs-dogs image classifier via transfer learning (beginner) · Sentiment classifier (intermediate) · Sequence model for time-series forecasting (advanced).

**⏱ Estimated time:** 80–140 hrs · 6–10 weeks

## Stage 10 — Big Data & Data Engineering

**Why it matters:** Data engineering skills let you handle large-scale data and build reliable pipelines feeding ML systems.

**Core topics:** Distributed computing with Spark (PySpark) · Hadoop basics · ETL pipelines · Workflow orchestration (Airflow) · Data lakes

| Resource | Platform | Duration | Level |
|---|---|---|---|
| Databricks free Spark courses 🥇 | Databricks | Self-paced | Beginner–Intermediate |
| Microsoft Learn — Data Engineering tracks 🥈 | Microsoft Learn | Self-paced | Beginner |

| [Apache Spark docs](https://spark.apache.org/docs/latest) / [Airflow docs](https://airflow.apache.org/docs) 📚 | Official documentation | Reference | Intermediate |

## 💼 Job-Ready Plan: Junior Data Analyst → Data Scientist

### Phase 1 — Junior Data Analyst (4–6 months)

**Skills:** Python + pandas · SQL (joins, aggregations, window functions) · Excel/Sheets · Visualization (Seaborn + Power BI/Tableau) · Basic statistics & EDA

**What to learn:** CS50P / Python for Everybody + freeCodeCamp Data Analysis → freeCodeCamp SQL + Kaggle SQL → Microsoft Learn Power BI → Khan Academy statistics

**Projects to build:**
1. Sales dashboard (Power BI/Tableau) on a public dataset
2. Customer segmentation with pandas, visualized in Seaborn/Plotly
3. Ad-hoc analysis notebook answering a real business question

**Certifications:** freeCodeCamp Data Analysis with Python certificate · optional Power BI Microsoft Learn badges

**Portfolio:** GitHub repo per project with a clear README and screenshots · dashboards published to Power BI Service / Tableau Public and linked

### Phase 2 — Junior Data Scientist (9–12+ months total)

**Skills:** Statistics & probability · Core ML + evaluation · Feature engineering & cross-validation · Basic deep learning (CNN/RNN) · Deployment (Docker + API)

**What to learn:** Google ML Crash Course + Kaggle ML micro-courses → DeepLearning.AI ML Specialization (audit) → Kaggle Feature Engineering + Inria MOOC → Fast.ai DL or freeCodeCamp ML with Python → ML Zoomcamp (deployment/MLOps)

**Projects to build:**
1. Churn prediction model with a full pipeline & feature engineering
2. Sales forecasting (time series) with a prediction dashboard
3. Recommendation system (books/movies) using collaborative filtering
4. One deep-learning project (image classifier or NLP sentiment model)
